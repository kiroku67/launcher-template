This is a template repository. To create your own launcher:

1. **Fork or download this repository**
2. **Customize the configuration:**
   - Update `docs/servers/ExampleServer-1.20.1/servermeta.json` with your server details
   - Update `docs/meta/distrometa.json` with your distribution metadata
   - Modify `launcher/package.json` with your branding
   - Update `admin-panel/main.js` with your repository info

3. **Generate distribution.json:**
   ```bash
   cd nebula
   npm install
   npm run start -- g distro distribution
   cp distribution.json ../docs/distribution.json
   ```

4. **Push to GitHub** and enable GitHub Pages (Settings â†’ Pages â†’ Source: main branch, /docs folder)

5. **Build the applications** or let GitHub Actions do it automatically

## ğŸ”§ Customization Guide

### 1. Server Configuration
Edit `docs/servers/YourServer/servermeta.json`:
```json
{
  "meta": {
    "name": "Your Server Name",
    "address": "your-server-ip:25565",
    "description": "Your description",
    "autoconnect": true
  },
  "forge": {
    "version": "47.3.0"
  }
}
```

### 2. Launcher Branding
Edit `launcher/package.json`:
```json
{
  "name": "yourserver-launcher",
  "productName": "Your Server Launcher",
  "author": "Your Name"
}
```

### 3. Distribution URL
Edit `launcher/app/assets/js/distromanager.js`:
```javascript
exports.REMOTE_DISTRO_URL = 'https://yourusername.github.io/yourrepo/distribution.json'
```

### 4. Admin Panel Configuration
Edit `admin-panel/main.js`:
```javascript
const REPO_OWNER = 'yourusername';
const REPO_NAME = 'yourrepo';
const SERVER_ID = 'YourServer-1.20.1';
```

## ğŸ“Š GitHub Actions Workflows

### Auto-Nebula Workflow
- **Trigger:** When files in `docs/servers/` are modified
- **Action:** Regenerates distribution.json automatically
- **Duration:** ~2-3 minutes

### Build Releases Workflow
- **Trigger:** When launcher or admin panel code changes
- **Action:** Builds executables for Windows, Mac, Linux
- **Duration:** ~10-15 minutes

### GitHub Pages Workflow
- **Trigger:** When `docs/` folder updates
- **Action:** Deploys distribution.json to public URL
- **Duration:** ~1-2 minutes

## ğŸŒ How It Works

1. **Mod Management:** Upload mods via Admin Panel â†’ Files pushed to GitHub
2. **Auto-Nebula:** GitHub Actions detects changes â†’ Regenerates distribution.json
3. **GitHub Pages:** Serves distribution.json publicly
4. **Launcher:** Downloads distribution.json â†’ Fetches mods/files â†’ Launches game
